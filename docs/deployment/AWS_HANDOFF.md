# AWS Handoff Checklist (EC2 Split + Jenkins)

Use this checklist for production handoff.

## 1. Architecture Confirmation

- Frontend hosted on dedicated EC2
- Backend API hosted on dedicated EC2
- PostgreSQL on Amazon RDS
- Product/Hero images stored in Amazon S3
- Domain on Route 53
- Jenkins used for CI/CD

## 2. DNS and Domain

- `www.example.com` -> frontend EC2
- `api.example.com` -> backend EC2
- HTTPS enabled (ACM + ALB/CloudFront or certbot on EC2)

## 3. Backend Runtime

- Node.js 20.x
- PM2 process manager installed
- Nginx reverse proxy configured
- App path: `/var/www/crackers-backend`

## 4. Backend Required Env

- `NODE_ENV=production`
- `PORT=3000`
- `SERVE_STATIC=false`
- `DATABASE_URL=...`
- `DB_SSL=true`
- `JWT_SECRET=...`
- `CORS_ALLOWED_ORIGINS=https://www.example.com`
- `AUTH_COOKIE_SAME_SITE=none` (cross-domain)
- `AUTH_COOKIE_SECURE=true`
- `STORAGE_DRIVER=s3`
- `S3_REGION=ap-south-1`
- `S3_BUCKET_NAME=your-assets-bucket`
- `S3_UPLOAD_PREFIX=uploads`
- `S3_PUBLIC_BASE_URL=https://your-assets-bucket.s3.ap-south-1.amazonaws.com` (optional)
- `S3_UPLOAD_ACL=` (optional, keep empty if bucket ACLs are disabled)

## 5. Frontend Runtime

File generated by Jenkins:

- `public/js/runtime-config.js`

Must contain:

- `API_BASE_URL=https://api.example.com`
- `ASSET_BASE_URL=https://api.example.com`
- `FRONTEND_BASE_URL=https://www.example.com`

## 6. Jenkins Requirements

- Jenkins credential id: `aws-ec2-ssh` (SSH private key)
- Pipeline file: root `Jenkinsfile`
- Jenkins agent has:
  - `node`, `npm`, `tar`, `ssh`, `scp`

## 7. Database

Run migrations:

1. `001_create_orders_tables.sql`
2. `002_create_user_addresses_table.sql`
3. `003_normalize_user_roles.sql`

Or use full SQL:

- `docs/database/DEVOPS_FULL_DATABASE_SCRIPT.sql`

## 8. S3 Checklist

- Bucket exists and public read is configured only for object path you need (or via CloudFront)
- Backend EC2 IAM role has `s3:PutObject` permissions on target bucket/prefix
- If using private bucket, `S3_PUBLIC_BASE_URL` points to CloudFront URL

## 9. Security Checklist

- RDS allows inbound only from backend SG
- Backend SSH restricted to Jenkins/admin IP
- Frontend SSH restricted to Jenkins/admin IP
- Secrets not stored in git
- `.env` created only on backend server

## 10. Post-Deployment Validation

- `GET https://api.example.com/api/health` -> `status: ok`
- Frontend loads on `https://www.example.com`
- Auth login/logout works with cookie
- Product/category/hero APIs work
- Admin pages work
- Uploads and product images load correctly
